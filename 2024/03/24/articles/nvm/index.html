A Survey of Software Techniques for Using Non-Volatile Memories for Storage and Main Memory Systems

本篇文章系统地列举了一遍针对应用在存储系统的NVM进行的一系列软件优化。

NVM基本介绍

什么是 Non-Volatile Memories

非易失性存储器（Non-Volatile Memories，NVM）是一种特殊类型的存储器，它在断电或关机后可以保持存储的数据不变，即数据是持久的。与易失性存储器（如随机存取存储器 RAM）不同，非易失性存储器不需要持续的电源供应来保持数据，因此具有持久性的特性。

非易失性存储器的一种常见形式是闪存存储器（Flash Memory），它被广泛应用于移动设备、数码相机、固态硬盘（SSD）等产品中。另外，还有一些新兴的非易失性存储器技术，如磁性随机存取存储器（Magnetoresistive Random Access Memory，MRAM）、相变存储器（Phase-Change Memory，PCM）、阻变存储器（Resistive Random-Access Memory，ReRAM）等，它们具有更快的读写速度、更高的存储密度和更低的功耗等优势。

常见的非易失性存储器（NVM）包括但不限于以下几种：

  1. 闪存存储器（Flash Memory）： 是目前应用最为广泛的一种非易失性存储器技术，分为 NOR Flash 和 NAND Flash 两种类型。NOR Flash 通常用于嵌入式系统中的代码存储，而 NAND Flash 则用于数据存储，如固态硬盘（SSD）、USB闪存驱动器、手机存储等。
  2. 磁性随机存取存储器（Magnetoresistive Random Access Memory，MRAM）： 利用磁性材料的磁性特性来存储数据，具有快速读写速度、低功耗和长寿命等优点，被视为一种有潜力替代传统存储器的新型技术。
  3. 相变存储器（Phase-Change Memory，PCM）： 利用相变材料的物理性质在不同的电阻状态间进行转变，实现数据的存储和读写操作，具有快速的读写速度和较高的存储密度。
  4. 阻变存储器（Resistive Random-Access Memory，ReRAM）： 通过改变存储单元的电阻状态来实现数据的存储和读写，具有快速的读写速度、低功耗和较高的存储密度。
  5. 存储级内存（Storage-Class Memory，SCM）： 是一类新型的存储器技术，包括三维堆叠存储器（3D XPoint）、STT-MRAM 等，具有介于内存和存储器之间的特性，可作为高速缓存、主存储器和持久性存储器使用。

而其中最主要的还是闪存，又分为nand flash和nor flash两种。
NOR Flash和NAND Flash是两种常见的闪存存储器类型，它们在结构、性能和应用方面有一些区别：

  1. 结构差异：
    • NOR Flash：NOR Flash 的存储单元是按字节（byte）寻址的，每个存储单元有一个地址，可以直接通过地址进行随机访问。因此，NOR Flash 适用于执行代码（如微处理器的引导代码）和需要快速随机读取的应用场景。
    • NAND Flash：NAND Flash 的存储单元是按块（block）寻址的，一般以页（page）为单位读写，以块为单位擦除。NAND Flash 通常用于大容量数据存储，如文件系统、媒体存储等场景。
  2. 性能差异：
    • NOR Flash：NOR Flash 具有较快的随机读取速度，适用于需要频繁随机访问的应用。但是，其写入和擦除速度相对较慢。【感觉就是新概念ROM（不是）】
    • NAND Flash：NAND Flash 具有较高的顺序读写速度和较低的成本，适用于大容量数据存储。但是，其随机读取速度相对较慢。【新概念磁盘（）】
  3. 应用场景：
    • NOR Flash：由于其较快的随机读取速度和适合执行代码的特性，NOR Flash 通常用于嵌入式系统中的引导程序、固件存储等场景。【比如某U盘】
    • NAND Flash：NAND Flash 通常用于数据存储，如固态硬盘（SSD）、USB闪存驱动器、SD存储卡、智能手机存储等。
  4. 耐用性：
    • NOR Flash：NOR Flash 的擦写次数通常比较高，因此具有较好的耐用性，适合于频繁写入和擦除的应用。
    • NAND Flash：NAND Flash 的擦写次数相对较低，但通过使用特定的管理技术（如wear leveling），可以延长其寿命。

总的来说，NOR Flash 和 NAND Flash 在结构、性能和应用方面有一定的差异，各自适用于不同的应用场景。NOR Flash 更适合于需要快速随机访问和频繁执行代码的场景，而NAND Flash 更适合于大容量数据存储和顺序读写的场景。

而在本篇文章中，着重介绍的是Flash（特指Nand Flash）、SCM（按字节寻址）以及PCM。

NVM管理优化技术

  1. Write/Erase Overhead Minimization

    尽量减小写入/擦除操作的overhead。具体来说，有以下几个思考方向：

    1. 预处理数据
      1. 减少数据量 增量写 / deduplication
      2. encoding data 减少’01’’10’pattern；使用extra bits
    2. 数据局部性 使用victim cache、write buffer，减少总体写入次数
    3. 针对特定介质优化 如对PCM，提前SET操作

  2. Wear-Leveling

    这部分也是大名鼎鼎的磨损平衡了，感觉各个方法都挺有意思的，可以看出磨损平衡的优化角度有两个，分配和迁移。

    1. 分配。以set为单位的固定分配、以logic block为单位的动态分配，或者是针对frequency分级的动态分配
    2. 迁移。迁移整个set，或者是lazy形式的remap

    不过总感觉这些方法都不是单独来看的，感觉都能一起用。。。比如说，我们可以采取一整个逻辑块的方法，然后根据上层文件系统的信息进行新老块的分配，再在擦除次数过多的时候进行remap操作或者migration操作

  3. Salvaging Faulty Blocks and Tolerating Failures

    大概就是讲发现故障和对应的处理方法，比如容忍一定程度的磨损、增强纠错码以更精确检查等，主要目的还是尽可能地提高寿命。

  4. Retention Relaxation

    主要讲的是对这个的一个trade-off：写越慢、擦写次数越少，留存时间越长。

  5. Flash I/O Scheduling

    也就是如题讲了一些IO调度方面的优化，如挂起写请求使读请求抢占、分读写区方便GC、划分多个区多个队列等，感觉也可以学习一下这种IO优化的思维

  6. Programming Models and APIs for Persistent Memory System Design && System-Level Redesign of NVM Storage and Memory

    大概从更软件更宏观的层面讲述了使用了NVM的系统进行的更多优化。

    比如说，Memorage就将nvm应用到了primary和secondary storage，感觉这也是primary和secondary storage都使用nvm的优势之一了，也即有种二者之间是没有boundary的感觉可以随便流动。
    并且这也引入了一个很有意思的虚拟化：secondary可以牺牲一部分容量作为primary的虚拟化，从而提高其整体的擦写寿命
